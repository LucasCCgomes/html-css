Historia Mascote do GITHUB

No ano de 2011, o GitHub decidiu que era hora de atualizar seu mascote. Ninguém na empresa se identificava como ilustrador, então chamaram C. McEfee para um pequeno projeto: ilustrar o Octocat fantasiado para as páginas de erro do site.

História

Cameron McEfee foi chamado para ilustrar o Octocat fantasiado para as páginas de erros do site que antes só mostravam o personagem sem muitas características pessoais.

Os fundadores sempre quiseram criar uma página de erro divertida para o GitHub, como foi muito difundido o estilo na época ( à la Google e seu dinossauro). O Git-software de controle de versão do qual o serviço se baseia- possui um conceito conhecido como “octopus merge”, no qual três ou mais ramos de desenvolvimento são combinados.

Os interessados compraram uma ilustração de um personagem meio gato, meio polvo em pé numa poça e colocaram-no em páginas que aparecem quando o servidor tem um problema ou o usuário solicita um link que não satisfaz. inicialmente, a personagem ficou conhecida como "Octopuss" ( que é quase o nome de um vilão da série 007). Ela foi criada por Simon Oxley ( designer de mascotes e logos como Twitter, GitHub, Bitly, Digital Ocean, iStock, TAM TAM Japan, Nulab Japan, etc.).
